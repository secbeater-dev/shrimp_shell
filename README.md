# 蝦殼買家統計分析網站

這是一個可直接部署到 GitHub Pages 的純前端分析工具。使用者匯入蝦殼購物紀錄 Excel 後，網站會在瀏覽器本機完成買家彙整、視覺化、篩選與匯出，不會把資料上傳到任何伺服器。

## 使用方式

1. 開啟 `index.html`。
2. 匯入符合蝦皮完整匯出格式，或 `Sample.xlsx` 這類簡化欄位格式的 Excel 檔。
3. 使用搜尋、日期、金額、次數、商品關鍵字篩選資料。
4. 匯出買家統計 Excel、統計 CSV 或目前篩選後的明細 CSV。

## 彙整規則

- 只分析 `COMPLETED` 訂單。
- 同一個 `買家蝦皮帳號` 彙整為同一列；簡化格式的 `買家帳號` 也會自動對應。
- `實際撥款額總額` 以唯一 `訂單編號` 加總一次，避免同訂單多商品列重複計價。
- `總購買次數` 為唯一完成訂單數。
- 商品以 `商品名稱 x 數量` 彙整，簡化格式的 `商品資訊` 也會自動對應。
- 數量優先由 `購買數量` 加總；若來源檔沒有 `購買數量` 欄位，則每筆資料列預設算 `1`。
- 電話與姓名會去重後以換行合併。

## 支援欄位

- 必要欄位：`訂單編號`、`訂單狀態`、`訂單成立時間`、`實際撥款額`、`買家蝦皮帳號`、`收件人姓名`、`收件人電話`、`商品名稱`。
- 可接受別名：`買家帳號`、`商品資訊`、`取件地址`、`賣家帳號`。
- 選用欄位：`購買數量`、`收件地址`、`賣家蝦皮帳號`、`付款方式`、`寄送方式`。

## GitHub Pages 部署

將整個資料夾內容推到 GitHub repository，並在 repository 的 Pages 設定選擇從根目錄發布即可。`vendor/` 內已包含必要的前端函式庫，不需要 npm build 或外部 CDN。
